# 개인용 대시보드 PRD

## 1. 개요

하루를 시작할 때 한 화면에서 필요한 정보를 모아 볼 수 있는 개인용 대시보드.
브라우저에서 여는 단일 페이지 웹앱이며, 고정 그리드 위에 위젯들이 배치된다.

- 한 줄 요약: "북마크 페이지처럼 가볍게 띄워두고, 한눈에 오늘을 파악한다."
- 사용자: 본인 1인 (인증/다중 사용자 없음)
- 기기: 데스크톱 브라우저 1대 기준 (다기기 동기화는 범위 밖)

## 2. 사용 범위 (Scope)

### In scope (1차 범위)
- 7개 위젯을 고정 그리드(4×2, 1칸 여백)로 배치
- 외부 API 2종(날씨, 환율) + 로컬 데이터 5종
- 브라우저 `localStorage` 기반 단일 기기 저장

### Out of scope (1차 범위 밖, 향후 확장 후보 — "미해결 의사결정" 참조)
- 다기기 동기화, 백엔드 서버, 사용자 인증
- 위젯 드래그/리사이즈 자유 배치
- 모바일 전용 레이아웃
- Google Calendar / ICS 연동
- 다크모드, 다국어(영문 UI)

## 3. UI / 레이아웃

- 형태: 브라우저 단일 페이지 웹앱 (`localhost:<port>` 로 접속)
- 레이아웃: **고정 그리드 4열 × 2행 = 8칸**, 위젯 7개 + 여백 1칸
- 권장 배치 (변경 가능):

  | (1,1) 시계 | (1,2) 날씨   | (1,3) 일정   | (1,4) 메모 |
  |------------|--------------|--------------|------------|
  | (2,1) 환율 | (2,2) 명언   | (2,3) 링크   | (여백)     |

- 톤: 미니멀 카드형. 위젯 박스에 제목 + 콘텐츠 + 갱신 시각(필요한 경우)

## 4. 위젯 명세

### 4.1 시계
- 데이터 소스: 시스템 시각
- 표시: `YYYY-MM-DD (요일)` + `HH:MM:SS`
- 갱신 주기: **1초**

### 4.2 날씨
- 데이터 소스: **Open-Meteo** (API 키 불필요, 무료)
- 기본 도시: **성남** (위경도 하드코딩 또는 설정 상수)
- 표시: 현재 기온, 체감 온도, 날씨 아이콘/설명, 최고/최저, 강수 확률
- 갱신 주기: **10분**
- 실패 정책: 마지막 성공 응답을 캐시에서 표시 + "갱신 실패" 표시

### 4.3 일정
- 데이터 소스: **로컬 직접 입력** (외부 캘린더 연동 없음)
- 데이터 모델: `{ id, date(YYYY-MM-DD), time(HH:MM | null), title, done }`
- 기능: 추가 / 삭제 / 완료 토글
- 표시 범위: **오늘 날짜의 일정**만 (시각 오름차순)
- 갱신 주기: **5분** (다른 탭/세션에서 변경 가능성 대비 `localStorage` 재읽기)
- 누적 전략: 과거 일정은 자동 삭제하지 않음 (사용자가 직접 삭제)

### 4.4 메모
- 형태: **체크 가능한 항목 리스트** (todo형 빠른 메모)
- 데이터 모델: `{ id, text, done, createdAt }`
- 기능: 추가 / 삭제 / 완료 토글
- 정렬: 최신 추가가 위, 완료된 항목은 하단으로 내리기
- 갱신: 사용자 인터랙션 즉시 반영 (별도 폴링 없음)
- 누적 전략: 자동 삭제 없음. 사용자가 정리

### 4.5 링크 (즐겨찾기)
- 형태: **1단계 섹션(카테고리) + 링크 리스트**
- 데이터 모델: `{ sectionId, name, links: [{ id, title, url }] }`
- 기능: 섹션 추가/이름변경/삭제, 링크 추가/편집/삭제
- 중복 처리: **URL 정규화** (`https://`/`http://` 통일, 끝 슬래시 제거, 쿼리/해시 보존) 후 같은 섹션 내 중복 추가 차단 + 안내 토스트
- 갱신: 사용자 인터랙션 즉시 반영

### 4.6 환율
- 데이터 소스: **exchangerate.host** (무료, 키 불필요)
- 통화쌍: **USD/JPY/EUR → KRW** 3종 (KRW 1원 환산이 아니라 1 USD/JPY/EUR 당 KRW)
- 표시: 통화별 현재 환율, 전일 대비 등락(↑/↓ + 차이)
- 갱신 주기: **30분**
- 실패 정책: 마지막 성공 응답 캐시 표시 + "갱신 실패" 표시

### 4.7 명언
- 데이터 소스: **정적 JSON 번들** (한국어 명언 모음, 코드와 함께 배포)
- 표시: 하루 1개 회전 (날짜 기반 인덱스 → 매일 자정 새 명언)
- 갱신 주기: **1일** (자정 통과 시 또는 페이지 로드 시 날짜 비교)
- 누적/이력: 사용자가 본 명언 이력 추적은 하지 않음

## 5. 데이터 저장

- 저장소: 브라우저 **`localStorage`**
  - 일정, 메모, 링크: 영구 저장
  - 위젯 설정(도시, 통화쌍 등): 향후 설정 UI 도입 시 사용. 1차에서는 코드 상수
- 외부 API 응답(날씨/환율): **메모리 캐시 + `localStorage` 백업** (오프라인/실패 시 마지막 값 표시)
- 키 네이밍 규칙: `dashboard.<widget>.<항목>` (예: `dashboard.memo.items`)
- 백업/내보내기: 1차 범위 밖

## 6. 수집/갱신 주기 요약

| 위젯 | 주기 | 비고 |
|---|---|---|
| 시계 | 1초 | `setInterval` |
| 일정 | 5분 | `localStorage` 재읽기 |
| 날씨 | 10분 | 외부 API |
| 환율 | 30분 | 외부 API |
| 명언 | 1일 | 날짜 변경 시 회전 |
| 메모 | 즉시 | 인터랙션 기반 |
| 링크 | 즉시 | 인터랙션 기반 |

## 7. 실패 / 에러 정책

- **외부 API 실패**(날씨/환율): 마지막 성공 응답을 표시, 위젯 우측 상단에 "갱신 실패 · 마지막 갱신 HH:MM" 노출. 다음 주기에 자동 재시도. 백오프/재시도 로직은 1차에서 단순화(다음 주기 재시도만).
- **localStorage 사용 불가**(시크릿 모드 등): 위젯은 인메모리로 동작하되 페이지 상단 1회 경고 배너 표시.
- **잘못된 입력**(빈 제목, 잘못된 URL): 추가 차단 + 인라인 에러 메시지.

## 8. 중복 처리 정책

- 링크: 같은 섹션 내 동일 URL(정규화 후) 추가 시 차단
- 메모: 중복 허용 (자유 입력)
- 일정: 중복 허용 (같은 시각/제목 가능)

## 9. 기술 스택

- **프레임워크**: React + Vite + TypeScript
- **상태 관리**: 위젯별 로컬 상태 + 공용 훅(`useLocalStorage`) 정도. 외부 라이브러리(Redux 등) 없이 시작
- **스타일**: CSS Modules 또는 Tailwind 중 구현 단계에서 결정 (미해결)
- **HTTP**: `fetch` 표준 사용
- **빌드/실행**: `npm run dev` (개발), `npm run build` (정적 산출물)
- **외부 의존**: Open-Meteo, exchangerate.host (둘 다 무료/무인증)

## 10. 디렉터리 구조 (제안, 코드 작성 단계에서 확정)

```
dashboard/

```

## 11. 성공 기준

- 페이지를 열었을 때 7개 위젯이 모두 렌더되고 1초 내 첫 콘텐츠 표시
- 외부 API가 죽어도 페이지 자체는 정상 동작 (캐시 또는 빈 상태 표시)
- 새로고침 후 메모/일정/링크가 그대로 복원됨
- 명시한 갱신 주기로 각 위젯이 자동 갱신됨

## 12. 1차 결정사항

> 본래 "미해결"로 분류했던 항목들을 1차 구현용 기본값으로 확정한다. 추후
> 변경하려면 본 문서를 갱신한다.

- **스타일링 방식**: **CSS Modules** 사용. (Vite 기본 지원, 추가 의존성/빌드
  설정 최소)
- **다크모드**: 1차 미지원. 향후 토글 형태 검토.
- **명언 데이터 출처/저작권**: **퍼블릭 도메인 한국어 명언 30~50개를 직접
  큐레이션**하여 `src/widgets/quote/quotes.ko.json` 정적 번들로 포함. 출처는
  국립국어원·고전 인용 등 저작권 만료 자료 위주.
- **환율 전일 비교**: **exchangerate.host `timeseries`** 엔드포인트 사용. 1회
  호출로 전일·당일 환율을 함께 받아 등락 계산.
- **날씨 위치 변경 UX**: 1차는 **성남 위경도 코드 상수 고정**. 설정 UI는
  out-of-scope.
- **시간대(Timezone)**: **시스템 로컬 시간** 사용. 강제 KST 옵션 미지원.
- **데이터 백업/내보내기**: 1차 범위 밖.
- **에러/이벤트 로깅**: **콘솔 로그만** 사용. 외부 수집 도구 도입 안 함.
- **테스트 전략**: **Vitest** 기반 단위 테스트만 도입. 대상은 (a) 스토리지
  어댑터/`useLocalStorage`, (b) URL 정규화, (c) 일정 필터(오늘만), (d) 명언
  날짜→인덱스 매핑, (e) 환율 등락 계산. UI 통합/E2E는 1차 제외.
