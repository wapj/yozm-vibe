# IMPL: T25. 전체 루프 통합 검증 + PRD 9번 성공 기준 확인 (M6 마감)

## 처리한 항목

M6의 마지막 태스크(T25). 새로운 프로덕션 기능은 추가하지 않고, 기존에 완성된
렌더·시뮬레이션·스토어·저장·React UI·효과음·스타일 경로가 PRD 9번 성공 기준
4개 축을 하나의 통합 지점(`App`)에서 실제로 함께 만족함을 신규/기존 테스트로
증명했다. 대조 중 프로덕션 코드의 실제 기능 간극은 발견되지 않았다(최소 변경
없음).

## 변경 파일

- `src/App.test.tsx`: 신규 `describe` 2건 추가.
  - **`T25: 전체 루프 통합 검증 (PRD 9번 성공 기준 1)`**: 주입 driver·
    `controllerOptions`(주입 timer·시드 rng)·`raceCanvasOverrides`(mock ctx·
    가짜 raf·시드 rng)로 `App`을 마운트해, 베팅 확정→카운트다운→경주 구동→
    완주→정산→(정산 표시 경과)→로비 복귀의 전체 순환을 `playRound()` 헬퍼로
    **연속 2회** 오류 없이 구동한다. 매 회차 종료 직후 `베팅 패널`이 다시
    보이고 `정산 결과`가 사라짐을 값으로 단언해, 1회차 정산 결과가 2회차
    시작을 막지 않음을 확인한다. 기존 `App.test.tsx`의 결정적 구동 헬퍼
    (`createInMemoryDriver`·`createManualTimer`·`createManualRaf`·
    `createMockCtx`·`driveRaceToCompletion`)를 그대로 재사용했다.
  - **`T25: 파산 자동 재충전 통합 검증 (PRD 4.4·9번 성공 기준 4)`**: 잔고
    150으로 시드한 `store`를 `App`에 주입하고 "올인"(잔고 전액) 베팅을
    확정하면, `handleBetConfirm`의 선차감(`adjustBalance(-amount)`) 시점에
    `gameStore` 내부 파산 판정이 **즉시** 발동해 잔고가 10,000원으로
    재충전되고 파산 횟수가 1회로 증가함을 화면 텍스트로 단언한다. 이어서
    카운트다운→경주 구동→완주→정산→로비 복귀까지 마저 구동해, 재충전 후에도
    전체 루프가 끊기지 않고 이어짐(파산 횟수 1회 유지, 베팅 패널 재노출)을
    확인한다.
- `docs/IMPL.md`(이 문서): 아래 PRD 9번 대조 표와 미검증 항목을 기록했다.

## 결정 사항

- **새 통합 테스트 배치**: 별도 파일을 만들지 않고 `src/App.test.tsx`에
  `describe` 블록만 추가했다. 이 파일이 이미 `App` 전체 마운트에 필요한 결정적
  구동 헬퍼(주입 driver·timer·raf·mock ctx)를 갖추고 있어, 신규 파일을 만들면
  동일한 헬퍼를 중복 정의해야 하는 비용이 더 크다(되돌리기 쉬운 배치 결정).
- **새로고침 라운드트립을 신규 테스트로 중복 구현하지 않음**: T22가 이미
  "라이프사이클 1회 구동 후 같은 driver로 새 `persistence.load()`를 호출해
  잔고·전적이 복원됨을 값으로 단언"하는 테스트를 종단 관점에서 보유하고
  있어(acceptance 문구와 정확히 일치), 동일한 테스트를 다시 작성하는 대신
  대조 표에서 인용으로 대체했다. 대신 "연속 2회 순환"은 T22에 없는 새로운
  축이므로 신규로 고정했다.
- **파산 재충전 시나리오는 "올인" 사용**: 파산 판정은 `adjustBalance` 내부에서
  선차감 시점에 즉시 발동한다(정산을 기다릴 필요 없음). 잔고 전액을 베팅해
  선차감만으로 잔고가 0이 되게 하는 것이 경주 결과(승/패)에 의존하지 않는
  가장 단순하고 결정적인 재현 경로다.

## 미검증 항목 (하네스 한계, 감점 아님)

jsdom은 `canvas.getContext("2d")`의 실제 그리기와 화면 픽셀을 확인할 수단이
없다. 아래는 PRD 9번 성공 기준의 일부이나 이 하네스로는 자동 검증할 수 없어
`npm run dev` 실브라우저 수동 확인 몫으로 남긴다:

- 역전 우승의 육안 체감 빈도(밴드 자체는 `driver.test.ts`로 고정, 체감은 별개).
- 슬로모션 감속의 부드러움, 폭죽 파티클의 밀도·화려함, 스킬 이펙트의 색·타이밍
  체감.
- 실황 자막의 노출 타이밍·가독성.
- 전반적 색 대비·여백 등 시각 완성도(T24가 세운 스타일 기준선의 실제 체감).
