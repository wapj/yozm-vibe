# 결정 주제: Canvas 렌더링 라이브러리

경주 화면(트랙, 말, 스킬 이펙트, 폭죽 파티클)을 어떤 렌더링 기술로 구현할지 결정한다.

## 후보

- A. Canvas 2D API 직접 사용 (외부 렌더링 라이브러리 없음)
- B. Pixi.js (WebGL 기반 2D 렌더링 라이브러리)

## 판단 기준

- 프로젝트 규모(출전마 4~8마리, 파티클 소량)에 비추어 적절한 표현력
- 번들 크기와 의존성 최소화
- 정적 호스팅만으로 배포 가능한 구성 유지
- 파티클·이펙트(오라, 스피드라인, 폭죽) 표현이 가능한 수준

합의 불발. 후보를 처음부터 다시 나열하고 장단점을 비교한 뒤 결정하라.

합의: A (시도 2회)
