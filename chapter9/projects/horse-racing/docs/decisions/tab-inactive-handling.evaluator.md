선택: B. 탭 비활성화 시 경주를 자동 일시정지, 복귀 시 재개

## 근거

PRD의 목표에 비추어 독립적으로 판단한다.

- 경주 결과의 신뢰성: 탭 비활성화 시 브라우저는 `requestAnimationFrame`을
  중단하거나 타이머를 스로틀한다. A(catch-up)는 복귀 시 누적된 경과 시간을
  한 번에 적분해 따라잡는데, 큰 dt를 단일 스텝으로 적분하면 Euler 적분 오차가
  커져 경주 결과가 왜곡될 수 있다. 실제로 이 프로젝트의 엔진은 명시적 Euler
  적분(`step`이 구간 시작 시점의 속도로 전진)이라 큰 dt에서 오차가 확대된다.
  B는 진행을 멈춰 이 왜곡 자체를 차단한다.
- delta-time 정합성: B는 pause 동안 dt 공급을 멈추는 것으로 충분하며,
  누적 dt 기반 시뮬레이션과 자연스럽게 정합한다.
- 구현 복잡도: A는 catch-up 시 큰 dt를 다시 작은 스텝으로 분할해야 오차를
  억제할 수 있어 추가 로직이 필요하다. B는 상태 머신에 pause/resume 전이만
  두면 되어 더 단순하다.

결과 신뢰성과 구현 단순성 모두에서 우위인 B를 선택한다.
