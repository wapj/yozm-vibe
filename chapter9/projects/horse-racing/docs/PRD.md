# PRD: 브라우저 경마 베팅 게임

## 1. 개요

브라우저에서 즐기는 싱글 플레이어 경마 베팅 게임입니다. 사용자는 출전마 중 한 마리에 가상 화폐를 베팅하고, 말들이 화면을 가로질러 달리는 경주를 관전합니다. 경주 결과에 따라 가상 잔고가 늘거나 줄어들며, 말마다 고유한 특성과 스킬이 있어 매 경주가 예측 불가능한 재미를 제공합니다.

- 대상 사용자: 가볍게 즐길 캐주얼 웹 게임을 원하는 일반 사용자
- 플랫폼: 데스크톱 브라우저 우선(모던 브라우저 최신 2개 버전). 모바일은 레이아웃이 깨지지 않는 수준으로만 대응
- 서버 없음: 완전한 클라이언트 사이드 애플리케이션. 정적 호스팅만으로 배포 가능

## 2. 기술 스택

- UI 프레임워크: **React 18+ + TypeScript + Vite**
- 경주 렌더링: **Canvas 2D** — 트랙, 말, 스킬 이펙트, 폭죽 등 애니메이션 요소는 Canvas에서 `requestAnimationFrame` 루프로 렌더링
- UI 레이어(베팅 패널, 잔고, 전적, 중계 텍스트)는 React 컴포넌트로 구성하고, Canvas와는 게임 상태 스토어를 통해 단방향으로 동기화
- 효과음: Web Audio API (외부 오디오 파일 없이 신시사이즈 또는 소형 에셋)
- 저장: `localStorage`
- 외부 연동: 없음 (네트워크 요청 없음)

## 3. 핵심 게임 루프

```
로비(베팅) → 카운트다운 → 경주(스킬 발동 가능) → 피니시 연출 → 정산 → 로비
```

1. **로비**: 출전마 목록과 각 말의 스탯·배당률·전적을 확인하고, 한 마리에 베팅 금액을 설정
2. **경주**: 말들이 좌→우로 트랙을 달림. 경주 중 각 말의 스킬이 확률적으로 발동
3. **피니시**: 선두 말이 결승선에 근접하면 전체 연출 속도가 슬로모션으로 감속되고, 통과 순간 폭죽 파티클 효과와 우승마 하이라이트 표시
4. **정산**: 베팅 결과에 따라 잔고 갱신, 전적 기록, 다음 회차를 위한 스탯 변동 적용

## 4. 기능 요구사항

### 4.1 출전마

- 기본 5마리 출전. 설정에서 **4~8마리** 범위로 조정 가능
- 각 말은 고유한 이름, 색상(또는 스프라이트), 성격 콘셉트를 가짐
- 스탯 구성:
  - `speed` (기본 속도): 평균 주행 속도
  - `stamina` (지구력): 후반 속도 유지력. 낮으면 종반에 감속
  - `burst` (순발력): 구간별 가속 변동 폭
  - `luck` (운): 스킬 발동 확률에 영향
- **회차별 스탯 변동**: 매 경주 시작 전 각 스탯에 ±10% 내외의 랜덤 변동을 적용. 변동 결과는 로비 화면에서 컨디션 지표(예: 화살표 ↑↓, 5단계 컨디션)로 사용자에게 노출하여 베팅 판단 자료로 제공

### 4.2 스킬 시스템 (일발역전 장치)

- 각 말은 고유 스킬 1개를 보유하며, 경주당 최대 1회 발동
- 발동은 자동(확률 기반)이며, 발동 확률은 `luck` 스탯과 현재 순위에 영향을 받음 — **하위권일수록 발동 확률 보정을 받아 역전 가능성 확보**
- 스킬 예시 (구현 시 5종 이상):
  - 라스트 스퍼트: 종반 구간에서 대폭 가속
  - 슬립스트림: 앞 말 바로 뒤에 있을 때 가속 지속
  - 스타트 대시: 출발 직후 폭발적 가속
  - 흔들기: 주변 말들을 일시 감속시킴
  - 무아지경: 일정 시간 스태미나 소모 없이 최고 속도 유지
- **스킬 발동 이펙트**: 발동 순간 해당 말에 시각 효과(오라, 잔상, 스피드라인, 화면 플래시 등)를 Canvas에 렌더링하고, 스킬명을 배너로 표시. 효과음 동반

### 4.3 베팅과 배당

- 베팅 방식: 단승식(1위 적중만 배당). 한 회차에 한 마리, 한 번만 베팅
- 베팅 금액: 최소 100 이상, 잔고 이하에서 프리셋 버튼(100/500/1,000/올인)과 직접 입력 지원
- **스탯 기반 변동 배당**: 매 회차 변동된 스탯으로 각 말의 우승 확률을 추정하고, 이를 역산해 배당률을 계산 (`배당률 ≈ 1 / 추정승률 × 하우스 계수`). 약체에 걸수록 고배당, 강자는 저배당
- 배당률은 로비 화면에 말별로 표시하며, 소수점 1자리(예: 3.2배)로 노출
- 정산: 적중 시 `베팅액 × 배당률`을 잔고에 지급, 미적중 시 베팅액 차감(베팅 시점에 선차감)

### 4.4 가상 잔고

- 시작 잔고: 10,000
- `localStorage`에 잔고·전적·설정을 저장하여 재방문 시 이어서 플레이
- **파산 처리**: 잔고가 최소 베팅액(100) 미만이 되면 기본 잔고(10,000)로 자동 재충전. 파산 횟수를 기록해 화면에 표시(가벼운 페널티 겸 재미 요소)
- localStorage 접근 불가(시크릿 모드 등) 시: 저장 없이 세션 메모리로만 동작하고, 저장이 비활성화되었음을 안내 배너로 표시

### 4.5 경주 연출

- 트랙: 가로 스크롤 없는 고정 화면. 좌측 출발 게이트, 우측 결승선
- 말 애니메이션: 달리기 모션(스프라이트 프레임 또는 다리 모션 그래픽), 순위 변동이 시각적으로 드러나는 속도 차
- 경주 시간: 회차당 약 8~15초
- **피니시 연출**:
  - 선두 말이 결승선 근접 시 전체 게임 속도를 슬로모션(예: 0.3배속)으로 감속
  - 결승선 통과 순간 폭죽 파티클 효과 + 우승마 스포트라이트
  - 접전(1~2위 근소 차) 시 "포토 피니시" 판정 연출 후 결과 공개
- 순위표: 경주 중 실시간 순위를 화면 한쪽에 표시

### 4.6 실황 중계 텍스트

- 경주 중 주요 이벤트를 자막형 중계 텍스트로 표시:
  - 출발("게이트가 열렸습니다!"), 선두 교체, 스킬 발동, 최종 직선 진입, 접전, 결승선 통과
- 이벤트 타입별 문구 풀에서 랜덤 선택하여 반복 플레이 시 단조로움 방지
- 최근 2~3줄을 유지하는 피드 형태로 표시

### 4.7 말 전적/성장 기록

- 말별 누적 기록: 출전 수, 우승 수, 승률, 최근 5경기 성적(예: 1-3-2-5-1)
- 로비의 말 카드에서 전적 열람 가능
- 연승 중인 말에는 "연승" 배지 표시
- 전적은 `localStorage`에 잔고와 함께 저장

### 4.8 효과음

- Web Audio 기반 효과음: 출발 팡파르, 달리는 발굽 소리(경주 중 루프), 스킬 발동음, 피니시 함성, 정산(승리/패배) 사운드
- 음소거 토글 제공. 토글 상태는 저장
- 브라우저 자동재생 정책 대응: 첫 사용자 입력(베팅 확정 등) 이후에만 AudioContext 활성화

### 4.9 설정

- 출전마 수 (4~8, 기본 5)
- 음소거 on/off
- 데이터 초기화(잔고·전적·설정 리셋, 확인 대화상자 필수)

## 5. 비기능 요구사항

- 60fps 목표의 부드러운 애니메이션 (`requestAnimationFrame` 기반, React 리렌더와 Canvas 렌더 루프 분리)
- 경주 로직(순위 산출, 스킬 발동, 배당 계산)은 렌더링과 분리된 순수 함수/모듈로 구현하여 단위 테스트 가능하게 유지
- 시드 없는 `Math.random()` 사용 가능하되, 경주 시뮬레이션 결과는 프레임레이트와 무관하게 결정되도록 시간 기반(delta time) 계산 적용
- 접근성: 색상만으로 말을 구분하지 않도록 번호·이름 병기

## 6. 실패 정책

- localStorage 저장 실패: 세션 메모리로 폴백, 안내 배너 표시 (4.4 참조)
- 저장 데이터 손상(파싱 실패 또는 스키마 불일치): 해당 데이터를 초기값으로 리셋하고 안내 표시. 앱이 중단되지 않아야 함
- 탭 비활성화 중 경주 진행: `requestAnimationFrame` 중단으로 인한 결과 왜곡을 막기 위해, 탭 비활성화 시 경주를 자동 일시정지하고 복귀 시 재개한다(자체 결정: 합의, 8번 참조)

## 7. 범위 제외 (Out of Scope)

- 실제 화폐·결제 연동
- 멀티플레이어, 서버, 계정 시스템
- 복승식·연승식 등 단승식 외 베팅 방식 (향후 확장 후보)
- 말 구매·육성 등 메타 게임

## 8. 미해결 의사결정

부트스트랩이 남긴 미해결 의사결정 3건은 `docs/decisions/`의 합의 절차를
거쳐 모두 확정되었다. 아래 결정을 확정 사항으로 반영한다. 미해결 항목은
남아 있지 않다.

- **Canvas 렌더링 라이브러리**: A. Canvas 2D API 직접 사용(외부 렌더링
  라이브러리 없음) (자체 결정: 합의). 근거 — PRD 2번의 Canvas 2D 명시,
  번들·의존성 최소화, 정적 호스팅 단순성. 4~8마리·소량 파티클 규모에 충분.
- **말 그래픽 표현**: A. 도형 + 다리 모션의 미니멀 그래픽(코드로 그리는
  벡터/도형) (자체 결정: 합의). 근거 — 외부 이미지 에셋 없이 정적 호스팅
  단순성 유지, 에셋 제작 비용 없음.
- **탭 비활성화 중 경주 처리**: B. 탭 비활성화 시 경주를 자동 일시정지하고
  복귀 시 재개 (자체 결정: 합의). 근거 — 결과 신뢰성(프레임레이트·경과
  시간 왜곡 방지)과 구현 단순성(PRD 5번, 6번 실패 정책 참조).

## 9. 성공 기준

- 베팅 → 경주 → 정산의 전체 루프가 오류 없이 반복 가능
- 스킬 발동으로 하위권 말이 역전 우승하는 사례가 체감 가능한 빈도(대략 5~10회차당 1회 수준)로 발생
- 피니시 슬로모션·폭죽, 스킬 이펙트, 실황 중계가 모두 동작
- 새로고침 후에도 잔고와 전적이 유지되고, 파산 시 자동 재충전이 동작
