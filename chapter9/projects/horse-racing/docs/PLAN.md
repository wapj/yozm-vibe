# PLAN: 브라우저 경마 베팅 게임

PRD.md를 기준으로 구현 계획을 정의한다. 현재 코드는 없으며(PRD와 이 문서만
존재), 첫 마일스톤부터 스캐폴딩을 시작한다.

## 아키텍처 원칙

- 게임 로직(스탯 변동, 배당 계산, 순위 산출, 스킬 발동, 시뮬레이션 적분)은
  렌더링·React와 분리된 순수 함수/모듈로 구현하여 단위 테스트 가능하게 유지한다.
- Canvas 렌더 루프(`requestAnimationFrame`)와 React 리렌더를 분리하고,
  게임 상태 스토어를 통해 단방향으로 동기화한다.
- 시뮬레이션은 delta-time(고정 스텝 적분) 기반으로 프레임레이트와 무관하게
  결정되도록 한다.

## 모듈 책임(예정)

- `src/domain/` — 타입, 스탯 변동, 승률 추정, 배당 계산 등 순수 로직
- `src/sim/` — 경주 시뮬레이션 엔진(순위, 스킬 발동, 피니시 판정)
- `src/store/` — 게임 상태 스토어와 상태 머신(로비→카운트다운→경주→피니시→정산)
- `src/persistence/` — localStorage 저장/로드, 손상·폴백 처리
- `src/render/` — Canvas 2D 렌더러(트랙, 말, 이펙트, 파티클, 순위표)
- `src/audio/` — Web Audio 효과음, 음소거
- `src/ui/` — React 컴포넌트(로비, 베팅 패널, 잔고, 전적, 중계, 설정, 배너)

## 미해결 의사결정 (합의 완료)

PRD 8번에서 이관했던 3건은 `docs/decisions/`의 합의 절차를 통해 모두 확정되어
PRD 8번에 (자체 결정: 합의) 표시와 함께 반영되었다. 미해결 항목은 없다.

| 주제 | 합의 | 구현에 반영될 마일스톤 |
|---|---|---|
| Canvas 렌더링 라이브러리 | A. Canvas 2D 직접 | M4 |
| 말 그래픽 표현 | A. 미니멀 도형 | M4 |
| 탭 비활성화 중 경주 처리 | B. 자동 일시정지 | M3 시뮬레이션 루프 / M2 상태 머신 |

- **Canvas 2D 직접**: M4 렌더러는 외부 렌더링 라이브러리(Pixi 등) 없이
  Canvas 2D API로 트랙·말·이펙트·파티클을 그린다.
- **미니멀 도형**: 말은 도형 + 다리 모션으로 렌더링한다. 외부 이미지 에셋을
  두지 않는다.
- **자동 일시정지**: 탭 비활성화(`visibilitychange`/`document.hidden`) 시
  경주를 일시정지하고 복귀 시 재개한다. 상태 머신에 pause/resume 전이가
  필요하므로 M2 상태 스토어 설계에 반영하고, M3 시뮬레이션 루프에서 실제로
  적용한다.
