# 경마 게임 PRD

## 1. 개요

브라우저에서 즐기는 가볍고 단순한 경마 게임. 사용자는 5마리 중 한 마리에
베팅하고, 약 10초간의 경주 결과에 따라 가상 잔고가 변동된다. 잔고가 0이
되면 게임 오버이며 다시 시작할 수 있다.

## 2. 목표 / 비목표

### 목표
- 한 화면에서 즉시 플레이 가능한 가벼운 경마 게임 제공
- 단승 베팅의 긴장감과 능력치 기반 오즈로 약간의 전략성 부여
- 외부 의존 없이 단일 HTML 파일 수준의 경량 구현

### 비목표
- 멀티플레이어, 서버 통신, 계정 시스템
- 실제 화폐, 실제 배당 시스템과의 연동
- 복잡한 베팅 종류(복승/연승/삼복승 등)
- 모바일 전용 최적화 (데스크톱 브라우저 우선)

## 3. 사용자 시나리오

1. 사용자가 페이지를 연다. 시작 잔고 1000이 표시된다.
2. 5마리의 말과 각 말의 동적 배당률이 화면에 표시된다.
3. 사용자가 한 마리를 선택하고 베팅 금액을 입력한 뒤 "출발" 버튼을 누른다.
4. 출발 신호 사운드가 울리고 약 10초간 말들이 트랙을 가로질러 달린다.
5. 결승선 통과 후 1등이 결정되고 결과 모달이 표시된다.
   - 베팅한 말이 1등이면: 잔고 += 베팅액 × 배당률
   - 그렇지 않으면: 잔고 -= 베팅액
6. 잔고가 1 이상이면 다음 경주로 진행, 0이면 게임 오버 화면 + "다시 시작" 버튼.

## 4. 기술 결정

| 항목 | 결정 |
|---|---|
| 형태 | 정적 단일 페이지 (HTML + CSS + JS), 외부 라이브러리 없음 |
| 그래픽 | 이모지 🐎 + 5개의 색깔 레인 |
| 사운드 | WebAudio API로 합성한 효과음 (외부 파일 없음) |
| 영속화 | `localStorage`로 잔고 유지 (새로고침해도 보존) |

## 5. 게임 규칙

### 5.1 출전과 트랙
- 출전 말: 5마리 (각각 고유한 이름/색상/이모지 색)
- 트랙: 가로 방향, 5개 레인. 출발선과 결승선이 명확히 표시됨.
- 1회 경주 소요 시간: 약 10초 (선두 기준)

### 5.2 능력치와 속도
- 매 경주 시작 시 각 말의 능력치(평균 속도, 변동성)가 새로 추첨된다.
- 매 프레임/틱마다 `속도 = 평균 + 노이즈` 로 위치를 갱신한다.
- 동률 방지: 결승선 통과 시 미세 노이즈를 추가하여 항상 1등이 1마리로 결정된다.

### 5.3 배당률 (오즈)
- 매 경주의 능력치 분포에 기반한 **동적 오즈**.
- 강한 말일수록 배당이 낮고(예: 1.5~2.5배), 약한 말일수록 배당이 높다(예: 5~10배).
- 오즈는 베팅 화면에 그대로 표시되어, 사용자가 능력치 차이를 간접적으로 인지할 수 있다.

### 5.4 베팅
- 종류: 단승만 (1등 맞히기)
- 시작 잔고: **1000**
- 최소 베팅: **10**
- 최대 베팅: **현재 잔고 전액**
- 베팅은 한 경주에 한 마리, 한 번만.

### 5.5 정산
- 적중: 잔고 += 베팅액 × 해당 말의 배당률 (베팅액은 차감 후 보상 지급, 즉 순이익 = 베팅액 × (배당률 - 1))
- 실패: 잔고 -= 베팅액

### 5.6 게임 오버 / 재시작
- 잔고가 0이 되면 게임 오버 화면 표시.
- "다시 시작" 버튼을 누르면 잔고가 1000으로 초기화되고 새 경주를 준비한다.

## 6. UI 흐름

1. **메인 화면**
   - 상단: 현재 잔고
   - 중앙: 트랙(5레인) + 5마리의 말 + 각 말의 이름과 배당률
   - 하단: 베팅 패널 (말 선택, 금액 입력, "출발" 버튼)
2. **경주 중 화면**
   - 베팅 패널 비활성화, 말들이 애니메이션으로 이동
3. **결과 모달**
   - 1~5위 순위, 사용자의 베팅 결과(획득/손실), "다음 경주" 버튼
4. **게임 오버 화면**
   - 메시지 + "다시 시작" 버튼

## 7. 사운드

WebAudio API로 합성:
- 출발 신호음 (짧은 비프 또는 종소리 톤)
- 경주 중 BGM 또는 발굽 리듬 (선택적, 단순 톤 루프)
- 결승선 도착 효과음
- 적중/실패 효과음

## 8. 데이터 / 영속화

`localStorage` 키:
- `balance`: 현재 잔고 (정수)
- `muted`: 음소거 상태 (문자열 `"1"` / `"0"`)
- 그 외 통계(누적 경주 수, 적중률 등)는 추후 결정 — 1차 구현에서는 위 두 키만 영속화한다.

## 9. 결정 사항 (1차 구현 기본값)

다음은 PRD 초안에서 미해결로 남았던 항목에 대한 1차 구현 기본값이다.
후속 사이클에서 사용자 피드백에 따라 조정될 수 있다.

### 9.1 말 5마리

| 이름 | 레인 색상 (Tailwind 톤) |
|---|---|
| Thunder | `#ef4444` (red-500) |
| Mystic | `#3b82f6` (blue-500) |
| Golden | `#f59e0b` (amber-500) |
| Emerald | `#10b981` (emerald-500) |
| Shadow | `#8b5cf6` (violet-500) |

이모지는 모든 레인에 동일하게 🐎 사용. 색상은 레인 배경/말 윤곽으로 표현.

### 9.2 능력치 분포 / 오즈 산식

- 평균 속도: 균등분포 `U(80, 120)` (정규화 단위/초)
- 매 틱 속도: `평균속도 × (1 + uniform(-0.2, 0.2))`
- 결승선 통과 시 미세 노이즈 `uniform(0, 0.001)` 가산하여 동률 방지
- 오즈 산식: `odds = round(clamp(SCALE / 평균속도, 1.5, 10.0), 2)`,
  `SCALE = 200`. 평균속도가 높을수록 배당이 낮아진다.
- 트랙 길이: 1000 정규화 단위(선두 기준 약 10초 도달).

### 9.3 트랙 시각

- 배경: 잔디톤 단색 또는 단순 가로 그라디언트
- 결승선: 검정/흰 체크 패턴 세로 띠
- 출발선: 단색 세로 띠

### 9.4 사운드

- BGM: **미사용** (1차)
- 효과음만: 출발 신호, 결승선 도착, 적중, 실패
- 우상단 음소거 토글 제공. 토글 상태는 `localStorage.muted`로 보존.

### 9.5 통계

- 1차 구현은 **잔고만 영속화**. 누적 경주 수 / 적중률 등은 미포함.
