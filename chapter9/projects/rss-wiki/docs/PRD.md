# PRD: rss-wiki

## 1. 개요

RSS 피드로 수집한 글을 LLM으로 요약하여 마크다운 위키 형태로 정리해 주는 개인용 도구. CLI와 로컬 웹 UI를 함께 제공한다.

사용자는 관심 있는 RSS 피드를 등록해 두고, 수집을 실행하면 새 글이 한국어로 요약되어 글 단위 마크다운 파일과 인덱스 페이지로 저장된다. 피드 등록·수집 실행·수집된 글 열람은 웹 UI에서도 수행할 수 있다.

## 2. 사용 범위

- 단일 사용자, 로컬 환경에서 사용하는 개인 도구.
- CLI는 명령을 실행할 때마다 동작하는 수동 실행 방식이다.
- 웹 UI는 로컬에서 실행하는 웹 서버(`rss-wiki serve`)로 제공하며, 본인 PC에서만 접근하는 것을 전제로 한다.
- 다중 사용자, 인증, 외부 배포는 범위에 포함하지 않는다.

## 3. UI 형태

### 3.1 CLI

서브커맨드 기반 인터페이스를 제공한다.

| 명령 | 동작 |
|---|---|
| `rss-wiki add <URL>` | 피드 등록 (피드 유효성 검증 포함) |
| `rss-wiki remove <URL 또는 이름>` | 피드 삭제 |
| `rss-wiki list` | 등록된 피드 목록 출력 |
| `rss-wiki fetch` | 전체 피드에서 새 글 수집 → 요약 → 위키 갱신 |
| `rss-wiki serve` | 로컬 웹 UI 서버 실행 |

- 피드 목록의 진실 소스는 도구가 관리하는 설정 파일이며, CLI와 웹 UI는 동일한 설정 파일·상태 파일을 공유한다.

### 3.2 웹 UI

`rss-wiki serve`로 실행하는 로컬 웹 애플리케이션. 다음 세 가지 작업을 웹에서 수행할 수 있다.

1. **피드 관리**: 피드 URL 등록(유효성 검증 포함), 삭제, 등록된 피드 목록 조회.
2. **수집 실행**: 버튼 클릭으로 `fetch`와 동일한 수집·요약·위키 갱신을 실행한다. 실행 중에는 피드별·글 단위 진행 상황을 실시간으로 표시하고, 완료 시 성공/실패 리포트를 보여 준다. 수집이 진행 중일 때 중복 실행은 차단한다.
3. **글 열람**: 수집된 글 요약을 웹에서 읽을 수 있다. 전체 최신순 목록, 피드별 목록, 날짜별 목록을 제공하며, 개별 글 페이지에는 요약문·핵심 포인트·원문 링크·발행일·피드명을 표시한다. 마크다운 산출물(`wiki/`)을 렌더링하여 보여 주므로 위키 파일과 웹 화면의 내용이 항상 일치한다.

### 3.3 디자인 품질 요구사항

웹 UI는 기능 동작만 확인하는 수준이 아니라, 완성도 있는 제품 수준의 디자인을 목표로 한다.

- **일관된 디자인 시스템**: 색상 팔레트, 타이포그래피, 간격(spacing) 스케일을 토큰으로 정의하고 전 화면에 일관되게 적용한다. 임의의 색상값·크기를 화면마다 다르게 사용하지 않는다.
- **읽기 경험 최우선**: 글 열람이 핵심 사용 흐름이므로, 본문 폭 제한(가독 폭 65~75자), 충분한 행간, 위계가 분명한 제목 체계 등 장문 읽기에 최적화한 타이포그래피를 적용한다.
- **다크 모드**: 라이트/다크 테마를 모두 지원하고 시스템 설정을 따르되 수동 전환도 제공한다.
- **반응형 레이아웃**: 데스크톱 기준으로 설계하되 브라우저 창 크기를 줄여도 레이아웃이 깨지지 않아야 한다.
- **상태 표현**: 로딩·빈 목록·오류 상태를 각각 명확한 화면(스켈레톤 또는 스피너, 빈 상태 안내, 오류 메시지)으로 표현한다. 아무 표시 없이 빈 화면을 보여 주지 않는다.
- **인터랙션 완성도**: 버튼·링크의 hover/focus/active 상태, 수집 진행률 표시, 부드러운 전환 등 세부 인터랙션을 다듬는다. 키보드 포커스 표시와 적절한 대비(WCAG AA)를 지킨다.

## 4. 핵심 기능

### 4.1 피드 수집
- 등록된 각 피드의 RSS/Atom을 파싱하여 새 글 목록을 얻는다.
- 피드 최초 등록 후 첫 수집 시 최신 10개까지만 수집한다. 개수는 옵션(`--limit`)으로 조정 가능하다.
- 이미 처리한 글은 상태 파일 기준으로 건너뛴다.

### 4.2 본문 확보
- RSS 항목의 링크로 원문 HTML을 가져와 본문을 추출한 뒤 요약 입력으로 사용한다.
- 원문 fetch 또는 본문 추출에 실패하면 RSS 항목의 description/content로 대체한다.

### 4.3 LLM 요약
- 로컬에 설치된 Claude Code CLI(`claude`)를 서브프로세스로 호출하여 요약을 생성한다. 별도 API 키 관리는 하지 않는다.
- 요약 형식: 원문 언어와 무관하게 **한국어**로, 요약문 3~5줄 + 핵심 포인트 불릿 목록.
- 요약 결과에는 원문 제목(원어), 원문 링크, 발행일, 피드명을 메타데이터로 포함한다.
- **병렬 실행**: 수집된 글들의 LLM 요약은 글 단위로 병렬 실행하여 전체 처리 시간을 단축한다.
  - 동시 실행 개수는 기본값(예: 4)을 두고 옵션(`--concurrency`)으로 조정 가능하게 한다.
  - 개별 글의 요약 실패는 다른 글의 처리에 영향을 주지 않는다(7장 실패 정책과 동일).
  - 병렬 처리 완료 후 위키 인덱스 갱신과 상태 파일 기록은 경합이 발생하지 않도록 직렬화하여 수행한다.

### 4.4 위키 생성
- 글 1개당 마크다운 파일 1개를 생성하고, 인덱스 페이지가 이를 연결한다.

```
wiki/
├─ index.md                     # 전체 인덱스 (최신 글, 피드 목록 링크)
├─ feeds/
│  └─ <피드-슬러그>.md           # 피드별 글 목록
├─ daily/
│  └─ YYYY-MM-DD.md             # 날짜(수집일)별 글 목록
└─ articles/
   └─ YYYY-MM-DD-<제목-슬러그>.md # 개별 글 요약
```

- `fetch` 실행이 끝날 때마다 인덱스(`index.md`, `feeds/*`, `daily/*`)를 재생성 또는 갱신한다.
- 파일명 슬러그는 제목을 기반으로 하되, 충돌 시 접미사(`-2` 등)를 붙인다.

## 5. 데이터 저장 방식

- 산출물: 마크다운 파일 (`wiki/` 디렉터리).
- 상태: `state.json` — 처리한 글의 식별자(URL 또는 GUID), 처리 시각, 실패 이력을 기록한다. 산출물과 상태를 분리하여 위키 파일을 수동으로 편집·삭제해도 중복 수집이 발생하지 않는다.
  - 매 `fetch`마다 인덱스(4.4)를 누적 재생성하기 위해, 처리한 각 글의 표시 메타(배정 파일명·제목·정규화 발행일·수집일·피드명)를 `state.json`에 함께 캐시한다. 파일명은 최초 배정 시 저장해 재생성 간 안정성을 보장한다. (자체 결정)
- 피드 목록: 도구가 관리하는 설정 파일(`feeds.json` 또는 유사 형식). CLI 명령으로만 조작한다.

## 6. 외부 연동

- **RSS/Atom 피드**: HTTP로 피드 XML을 조회한다.
- **원문 페이지**: HTTP로 글 원문 HTML을 조회하고 본문을 추출한다.
- **Claude Code CLI**: 요약 생성용. `claude` 명령이 PATH에 있어야 하며, 없으면 명확한 오류 메시지와 함께 중단한다.
- 그 외 외부 서비스 연동(알림, 배포 등)은 범위 밖이다.

## 7. 실패 정책

- 기본 원칙: **건너뛰고 계속 진행 + 말미 리포트**.
  - 피드 단위 실패(네트워크 오류, 파싱 불가): 해당 피드를 건너뛰고 다음 피드를 처리한다.
  - 글 단위 실패(원문 fetch 실패 → RSS 본문 대체도 불가, LLM 요약 실패): 해당 글을 건너뛰고 다음 글을 처리한다.
- 실패한 글은 상태 파일에 "처리 완료"로 기록하지 않아, 다음 `fetch` 실행에서 자동으로 재시도된다.
- 실행 종료 시 성공/실패 건수와 실패 사유를 요약한 리포트를 표준 출력으로 보여 준다.
- 부분 실패는 종료 코드 0(경고 포함), 전체 실패 또는 실행 불가(예: `claude` 미설치)는 0이 아닌 종료 코드를 반환한다.

## 8. 기술 스택

- **언어**: Python 3.12+
- **패키지 관리**: uv
- **CLI 프레임워크**: Typer
- **RSS 파싱**: feedparser
- **본문 추출**: trafilatura (또는 readability 계열)
- **LLM**: Claude Code CLI 서브프로세스 호출 (`claude -p` 비대화 모드), `asyncio` 기반 병렬 실행 (세마포어로 동시 실행 개수 제한)
- **웹 서버**: FastAPI + Uvicorn (로컬 실행 전용)
- **웹 프런트엔드**: 서버 사이드 렌더링(Jinja2 템플릿) 중심으로 구성하되, 수집 진행 상황 등 실시간 갱신이 필요한 부분은 경량 JavaScript(SSE 또는 폴링)로 처리

## 9. 비목표 (Non-goals)

- 외부 배포용 서버, 상시 실행 데몬, 스케줄러 내장 (웹 UI는 로컬 실행 전용)
- 다중 사용자 지원, 인증
- 전문 검색(full-text search) 기능
- OPML 가져오기/내보내기 (초기 버전에서는 제외)
