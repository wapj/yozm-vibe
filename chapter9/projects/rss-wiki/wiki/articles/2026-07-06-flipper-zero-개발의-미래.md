## 요약

Flipper Devices는 공식 Flipper Zero 펌웨어 개발이 중단됐다는 커뮤니티의 우려에 대응하여, 펌웨어 유지보수와 커뮤니티 기여 지원에 다시 리소스를 배정했습니다. 다만 팀의 주력은 새 기기 개발이므로, 소통은 GitHub Discussions의 투표 기반 기능 요청으로 일원화하고 실시간 소통은 중단합니다. 700KB 플래시 메모리 제약을 microSD 기반 동적 앱 로딩 구조로 우회한 것이 2024년 stable firmware 1.0의 기반이 되었고, 팀은 Kickstarter 약속과 개발 목표가 모두 달성됐다고 평가합니다. 앞으로 pull request는 강화된 기여 가이드에 따라 엄격히 심사되며, 모든 펌웨어 변경에 통합·회귀 테스트가 요구됩니다.

## 핵심 포인트

- **유지보수 재개**: 커뮤니티 반발 이후 펌웨어 유지보수에 리소스를 재배정하고, 커뮤니티 요청을 매주 검토하기로 함
- **소통 방식 변경**: 개발팀과의 소통은 GitHub Discussions의 기능 요청으로만 진행하며, 투표 수가 많은 요청을 우선 검토. 일반 토론·질문은 Discord, Reddit 등으로 분리
- **아키텍처 전환**: 펌웨어용 플래시가 700KB뿐인 하드웨어 제약 때문에 microSD에서 앱을 동적으로 로드하는 구조를 도입했고, 이것이 stable firmware 1.0과 안정화된 API/SDK의 토대가 됨
- **기여 심사 강화**: 검증이 어려운 AI 생성 코드, 저수준 라이브러리 변경, UI·문서에 영향을 주는 변경은 특히 엄격히 평가되며, QA 팀의 테스트 케이스를 공개해 커뮤니티도 회귀 테스트에 참여
- **커뮤니티 반응**: Hacker News에서는 "일회성 하드웨어 판매 구조에서 지속적 펌웨어 업데이트 기대는 무리"라는 지적, 공식 펌웨어보다 Momentum·Unleashed 같은 대체 펌웨어를 선호한다는 의견, 공식 Discord의 폐쇄적 운영에 대한 불만 등이 제기됨. 반면 실사용자들은 RFID 복사, 적외선 리모컨 등 실용성을 높게 평가
