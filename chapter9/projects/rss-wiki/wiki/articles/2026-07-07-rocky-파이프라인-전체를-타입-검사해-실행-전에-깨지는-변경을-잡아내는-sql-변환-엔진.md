# Rocky - 파이프라인 전체를 타입 검사해 실행 전에 깨지는 변경을 잡아내는 SQL 변환 엔진

- 발행일: 2026-07-07
- 피드: GeekNews - 개발/기술/스타트업 뉴스 서비스

## 요약

Rocky는 SQL 변환 파이프라인 전체를 실행 전에 타입 검사하여, 운영 환경에서 실패할 변경을 미리 차단하는 오픈소스(Apache 2.0) 데이터 변환 엔진입니다. 상류 스키마 변경(스키마 드리프트)을 자동 감지해 타겟을 재생성하고, 필수 컬럼 누락이나 위험한 타입 변경 같은 데이터 계약 위반을 프로덕션 테이블에 쓰기 전에 에러 코드로 잡아냅니다. 컬럼 단위 계보 추적, 브랜치 기반 격리 실험 환경, 자연어 기반 AI 모델 생성, VS Code language server 등 개발 편의 기능을 Rust 바이너리 하나로 제공합니다. 기존 웨어하우스(Databricks, Snowflake, BigQuery 등)를 그대로 유지하면서 사용할 수 있고, 순수 SQL 렌더링과 dbt 프로젝트 변환 명령을 제공하여 벤더 락인이 없습니다.

## 핵심 포인트

- **컴파일 타임 검증**: 파이프라인 전체 타입 검사로 "개발에선 통과, 운영에서 실패"하는 쿼리를 실행 전에 차단하고, 데이터 계약 위반을 에러 코드(E010, E013)로 보고
- **스키마 드리프트 자동 감지**: 소스 테이블의 컬럼 타입/이름 변경을 diff로 감지하고 타겟을 자동 재생성
- **컬럼 단위 계보(column lineage)**: 특정 컬럼을 시드부터 최종 팩트 테이블까지 역추적 가능
- **브랜치 격리**: `rocky branch create`로 별도 스키마에 실험 환경을 만들고, 확인 후 프로모션 또는 드롭
- **AI 모델 생성**: 자연어 입력으로 Rocky DSL 모델을 자동 생성하며, 컴파일 실패 시 자동 재시도
- **개발 도구**: VS Code language server로 작성 중 타입 불일치·깨진 참조 표시, go-to-definition 지원
- **운영 기능**: PII 태그 기반 환경별 데이터 마스킹, `strategy = "incremental"` 설정만으로 워터마크 기반 인크리멘탈 로드 자동 관리
- **벤더 락인 없음**: `rocky emit-sql`로 오프라인에서 의존성 정렬된 순수 SQL 렌더링, `rocky import-dbt`로 dbt Core 프로젝트 일괄 변환
- **지원 범위**: 어댑터는 Databricks, Snowflake, BigQuery, Trino, DuckDB / 소스는 Fivetran, Airbyte, Iceberg, Manual / Apache 2.0 라이선스
