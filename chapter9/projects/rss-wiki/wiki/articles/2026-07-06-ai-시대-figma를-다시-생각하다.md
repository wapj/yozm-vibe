# AI 시대, Figma를 다시 생각하다

- 원문: [https://news.hada.io/topic?id=31169](https://news.hada.io/topic?id=31169)
- 발행일: 2026-07-06
- 피드: GeekNews - 개발/기술/스타트업 뉴스 서비스

## 요약

Config 2026에서 Figma는 Code layers, Motion, 셰이더, 에이전트 워크플로 등을 발표하며 AI 시대의 압박에 대응하는 승부수를 던졌습니다. 10여 년간 제품 디자인의 중심이었던 캔버스는 AI가 기획부터 출시까지의 워크플로를 코드 에디터와 에이전트 환경으로 옮기면서 그 중심성을 위협받고 있습니다. 특히 Anthropic의 Claude Code와 Claude Design은 의도를 곧바로 코드 실행으로 연결해 정적 목업 단계를 생략하는 방식을 정상화하며, Figma의 좌석 기반 수익 모델과 "기록 원천으로서의 디자인 파일"이라는 전제를 동시에 잠식하고 있습니다. Figma는 캔버스 개선을 넘어 도구 간 디자인 의도와 구현 정렬을 보존하는 오퍼레이팅 레이어로의 전환을 시도하고 있으며, 진짜 시험대는 캔버스가 중심에서 물러난 뒤에도 디자인이 살아남게 하는 것입니다. 승패는 컨퍼런스 데모가 아니라 팀들이 실제로 어디에서 시간을 보내는지에 따라 판가름 납니다.

## 핵심 포인트

- **Config 2026의 신기능**: Code layers(코드를 캔버스로), Figma Motion(애니메이션 편입), 셰이더 도구, 에이전트 워크플로 강화 — 캔버스를 방어하기 위한 확장 전략입니다.
- **좌석(seat) 모델의 균열**: 엔지니어가 코딩 환경에서 UI를 직접 생성·검수할 수 있게 되면서, 검토·참조용 수동 좌석의 필요성이 약화되고 있습니다.
- **Anthropic의 위협**: Claude Code와 Claude Design의 결합은 캔버스를 정적 목업이 아닌 소프트웨어 생성용 라이브 인터페이스로 전환하며, "드로잉을 덜 중심적으로 만드는 워크플로"라는 가장 강력한 위협을 제기합니다.
- **이식성의 시대**: Photoshop → Sketch → Figma로 이어진 역사처럼 독점 도구는 시장이 이식성을 요구하는 순간까지만 강력하며, 미래의 자산은 시각 파일이 아니라 구조화된 재사용 가능한 의도(intent)입니다.
- **진짜 시험대**: 팀이 핵심 시간을 Figma 안에서 보내면 캔버스가 지배력을 유지하지만, 코드 에디터·AI 에이전트·도구 간을 오가는 구조화된 디자인 시스템으로 이동하면 Figma는 여러 플랫폼 중 하나로 전락합니다.
- **관점의 전환**: 제품 디자인이 캔버스 우선 분야로 남을지, 아니면 캔버스가 가끔만 필요한 "의도의 아키텍처"가 될지가 이 전환의 핵심 질문입니다.
