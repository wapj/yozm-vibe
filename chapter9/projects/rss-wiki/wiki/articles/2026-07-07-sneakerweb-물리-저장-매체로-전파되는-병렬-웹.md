# sneakerweb - 물리 저장 매체로 전파되는 병렬 웹

- 원문: [https://news.hada.io/topic?id=31214](https://news.hada.io/topic?id=31214)
- 발행일: 2026-07-07
- 피드: GeekNews - 개발/기술/스타트업 뉴스 서비스

## 요약

sneakerweb은 DNS 서버, 도메인 등록기관, 웹 호스트 없이 웹사이트를 게시하고 전달하는 것을 목표로 하는 P2P 프로토콜입니다. 웹사이트 데이터는 사용자 기기에 직접 저장되며, 네트워크 전송 대신 USB 등 물리 저장 매체를 통해 `.snk` 파일 형태로 다른 사용자에게 전달됩니다. 수집한 웹사이트는 평소 사용하는 브라우저에서 오프라인으로 열람할 수 있고, CLI를 통해 도메인 클레임, 게시, 가져오기·내보내기, 차단, 로컬 서빙 등의 작업을 수행합니다. 프로토콜은 Willow를 기반으로 하여 웹사이트 위조 방지, 업데이트 병합, 작고 효율적인 `.snk` 파일 생성을 처리합니다. 커뮤니티에서는 스팸·사기 확산 가능성에 대한 우려가 제기되었으나, 물리 매체 교환의 느린 전파 속도와 높은 마찰 때문에 대규모 악용 대상이 되기는 어렵다는 반론이 있었습니다.

## 핵심 포인트

- **탈중앙 웹 게시**: DNS, 도메인 등록기관, 웹 호스트 없이 웹사이트를 게시하는 P2P 프로토콜입니다.
- **물리 매체 전파**: 사이트 데이터는 기기에 직접 저장되고, 네트워크가 아닌 물리 저장 매체로 `.snk` 파일을 옮겨 공유합니다.
- **오프라인 열람**: 수집한 웹사이트는 일반 웹 브라우저에서 오프라인으로 열람할 수 있습니다.
- **CLI 제공**: `domain`(도메인 클레임), `publish`(게시), `export`/`import`(`.snk` 내보내기·가져오기), `block`(도메인 차단), `serve`(로컬 서버) 명령을 지원합니다.
- **Willow 기반 프로토콜**: 웹사이트 위조 방지, 업데이트 데이터 병합, 효율적인 `.snk` 파일 생성을 프로토콜 수준에서 처리합니다.
- **커뮤니티 반응**: Willow·Bab 제작진의 프로젝트라는 점에 대한 호평과 함께, 스팸 우려에 대해서는 전파 속도가 느리고 마찰이 커서 대규모 문제로 확산될 가능성이 낮다는 의견이 나왔습니다.
