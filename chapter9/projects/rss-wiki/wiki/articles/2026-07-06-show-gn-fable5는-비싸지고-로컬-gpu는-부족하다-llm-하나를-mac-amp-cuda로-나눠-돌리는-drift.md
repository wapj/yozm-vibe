- 피드: GeekNews - 개발/기술/스타트업 뉴스 서비스

## 요약

DRIFT는 하나의 LLM을 여러 개인 기기에 디코더 레이어 단위로 분할해 실행하는 오픈소스 프로젝트입니다. 최상위 모델의 접근이 제한되고 로컬 모델도 높은 GPU 사양을 요구하는 상황에서, Mac의 Apple GPU(MPS)와 NVIDIA GPU(CUDA)를 혼합해 하나의 모델을 함께 실행하는 방식을 제안합니다. 노드 간에는 hidden state만 전송하며, 통신은 특정 ML 런타임에 의존하지 않는 TCP + msgpack 기반의 중립적 바이트 프로토콜로 처리합니다. 속도보다 정확성을 우선순위로 두어, 분산 실행 결과가 단일 머신 실행 결과와 동일한지 검증하는 데 무게를 둡니다. Exo나 llama.cpp RPC와 달리 특정 생태계나 런타임에 종속되지 않는다는 점이 핵심 차별점입니다.

## 핵심 포인트

- **문제 의식**: 프론티어 모델(Fable5 등)은 개인 접근이 제한되고, 로컬 모델은 높은 GPU 메모리를 요구해 AI 실행 능력이 일부 하드웨어·플랫폼에 집중되고 있음
- **핵심 아이디어**: 모델을 디코더 레이어 단위로 자동 분할하고, 노드 간에는 모델 전체나 KV 캐시가 아닌 hidden state만 전송
- **이기종 혼합 실행**: Apple MPS와 NVIDIA CUDA를 하나의 모델 실행에 함께 사용 — 원래 같은 분산 런타임에 넣기 어려운 조합
- **신뢰성 기능**: X25519·ChaCha20-Poly1305 암호화, Ed25519 서명 영수증으로 노드 작업 검증, 노드 장애 시 re-split + replay 기반 failover
- **호환성·규모**: OpenAI 호환 HTTP API 제공, 이론상 Qwen 기준 최대 28대 분할 가능하나 현실적 스위트 스팟은 2~4대
- **차별점**: Exo(Apple 생태계 종속)·llama.cpp RPC(ggml 런타임 종속)와 달리 노드 간 경계를 중립적 바이트 프로토콜로 정의
- **설계 철학**: 토큰 경제 같은 거대 담론이 아니라, 검증 가능하고 복구 가능한 실행 계층부터 만드는 것이 AI 탈중앙화의 출발점이라는 관점
