# PRD: 뽀모도로 할일 관리 웹 도구 (todo-app)

## 1. 개요

할일 목록을 관리하면서 각 할일에 뽀모도로 타이머를 연결하여 집중 시간을 기록하는 웹 도구입니다. 할일을 클릭하면 25분 집중 타이머가 시작되고, 완료된 뽀모도로 횟수가 할일별로 누적 표시됩니다.

- 사용 범위: 개인용, 단일 사용자, 로컬 브라우저에서 동작
- 배포 형태: 정적 웹 앱 (백엔드 없음)

## 2. 기술 스택

- Vite + React + TypeScript
- 단위 테스트: Vitest
- 상태 관리: React 내장 기능만 사용 (외부 상태 라이브러리 미사용)
- 데이터 저장: 브라우저 `localStorage`

## 3. 핵심 기능

### 3.1 할일 관리

- 할일 추가 / 수정 / 삭제 / 완료 체크
- 할일에 태그를 여러 개 부착 가능
- 태그 기준 필터링 (선택한 태그의 할일만 표시)
- 각 할일 옆에 완료된 뽀모도로 누적 횟수 표시

### 3.2 뽀모도로 타이머

- 할일을 클릭하면 해당 할일에 대한 25분 집중 타이머 시작
- 동시에 하나의 타이머만 진행 가능 (다른 할일 클릭 시 확인 후 기존 타이머 중단)
- 집중 25분 완료 시 해당 할일의 뽀모도로 카운트 +1
- 중간 중단 시 카운트에 포함하지 않으며, 타임라인에 "중단" 기록으로 남김

### 3.3 휴식 타이머

- 집중 완료 후 5분 짧은 휴식 안내
- 뽀모도로 4회 완료마다 15분 긴 휴식 제안
- 휴식은 강제하지 않으며 사용자가 건너뛸 수 있음

### 3.4 기록 / 타임라인

- 각 뽀모도로의 시작/종료 시각, 대상 할일, 결과(완료/중단)를 기록
- 타임라인 화면에서 이력 목록을 시간순으로 조회

### 3.5 알림

- 타이머 종료 시 화면 내 알림(배너 또는 모달) 표시
- 타이머 진행 중 브라우저 탭 제목에 남은 시간 표시 (예: `(12:34) todo-app`)
- 소리 알림, OS Notification API는 범위에 포함하지 않음

## 4. 데이터 저장 방식

- 모든 데이터(할일, 태그, 뽀모도로 기록, 진행 중 타이머 상태)를 `localStorage`에 저장
- 저장 데이터에 스키마 버전 필드를 포함하여 향후 마이그레이션에 대비

### 데이터 모델 (초안)

```
Todo      { id, title, tags: string[], done, createdAt }
Session   { id, todoId, type: 'focus' | 'shortBreak' | 'longBreak',
            startedAt, endedAt, result: 'completed' | 'aborted' }
TimerState{ todoId, type, startedAt }   // 진행 중 타이머 (복원용)
```

## 5. 실패 정책

- 타이머 진행 중 새로고침/브라우저 종료: 시작 시각(`startedAt`)을 저장해 두고, 재접속 시 경과 시간을 계산하여 타이머를 이어서 표시. 이미 25분이 경과했으면 완료로 기록
- `localStorage` 읽기 실패 또는 데이터 파싱 실패: 손상된 항목은 무시하고 빈 상태로 시작하되, 화면에 경고 배너로 안내
- `localStorage` 쓰기 실패(용량 초과 등): 화면에 경고를 표시하고 메모리 상태로는 계속 동작
