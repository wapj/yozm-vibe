# PRD — Todo + Pomodoro Web App

## 1. 개요

할일 목록을 관리하면서 각 할일에 뽀모도로 타이머를 붙여 집중 시간을 기록하는 1인용 웹 도구. 할일을 클릭하면 25분 타이머가 시작되고, 완료한 뽀모도로 횟수가 할일별로 누적된다.

## 2. 사용자 및 사용 범위

- **대상**: 단일 사용자(본인) 1인용
- **인증**: 없음 (로컬에서 단독 실행)
- **동시 사용**: 단일 브라우저 세션 기준. 멀티 탭/멀티 디바이스 동기화는 비목표

## 3. 형태 및 저장소

- **UI**: 단일 페이지 앱(SPA). 타이머 UX가 매끄럽도록 클라이언트 사이드 렌더링
- **백엔드**: 가벼운 API 서버 (할일 CRUD, 뽀모도로 세션 기록, 통계 집계)
- **저장소**: SQLite 단일 파일
  - 이유: 백업/이동 용이, 통계 쿼리 작성 쉬움, 1인용에 충분
  - 스키마는 구현 단계에서 정의 (테이블 예: `tasks`, `tags`, `task_tags`, `pomodoro_sessions`, `settings`)

## 4. 할일(Task) 기능

### 4.1 속성
- 제목 (필수)
- 메모 (선택, 짧은 텍스트)
- 우선순위: 높음 / 보통 / 낮음 (기본: 보통, 정렬용)
- 태그: 사용자 정의, 다중 부여 가능
- 상태: 진행중 / 완료
- 누적 뽀모도로 횟수 (read-only, 세션 기록에서 집계)
- 생성/수정/완료 시각

### 4.2 라이프사이클
- 생성, 편집, 삭제 가능
- 완료 처리한 할일은 기본 숨김 → "완료된 항목 보기" 토글로 펼침
- 삭제는 소프트 삭제 권장 (통계 집계 보존). 단, 영구 삭제도 가능

### 4.3 카테고리/태그
- **고정 카테고리 없음**. 사용자가 자유롭게 태그 생성
- 할일 하나에 태그 여러 개 부여 가능
- 태그는 별도 관리 화면 없이 입력 시 자동 생성/재사용

## 5. 검색 및 필터

- **키워드 검색**: 제목 + 메모 대상 부분 일치
- **태그 필터**: 다중 선택 (AND 조건)
- **날짜 프리셋**: 오늘 / 이번 주 / 전체
  - 기준: 할일 생성일 또는 마지막 활동(뽀모도로) 시각 — 구현 시 후자가 자연스러움
- **상태 필터**: 진행중 / 완료(토글)

## 6. 뽀모도로 타이머

### 6.1 규칙 (정통 뽀모도로)
- 집중: 25분
- 짧은 휴식: 5분
- 긴 휴식: 15분 (4세션마다)
- 모든 시간 길이는 설정 화면에서 조정 가능

### 6.2 동작
- 할일 카드를 클릭하면 해당 할일에 대한 25분 집중 세션이 시작됨
- 세션 진행 중에는 카운트다운 + 진행률 표시
- 25분이 끝나면 자동으로 휴식 모드로 전환
- 휴식이 끝나면 다음 집중 세션을 시작할지 묻거나 자동 시작 (설정 가능, 기본: 묻기)

### 6.3 동시 실행
- **단일 세션만 허용**. 한 번에 한 할일만 타이머 가능
- 다른 할일을 클릭하면 다이얼로그: "현재 세션을 (1) 완료로 처리 / (2) 폐기 / (3) 취소"

### 6.4 상태 복원
- 세션 시작 시각, 길이, 단계(집중/휴식)를 서버에 저장
- 새로고침/창 닫기 후 재방문 시 경과 시간을 계산하여 타이머 상태 복원
- 경과 시간이 세션 길이를 넘었다면 자동으로 다음 단계로 이동

### 6.5 알림
- 세션 종료 시 **브라우저 알림 + 소리** 두 가지 동시 발화
- 소리는 설정에서 끄기 가능
- 브라우저 알림 권한이 없으면 페이지 내 토스트로 대체

### 6.6 카운트 정책
- **정상 종료(25분 완주)**: 해당 할일의 누적 뽀모도로에 +1
- **중도 포기/폐기**: 누적에는 미포함, 단 별도 "포기 횟수" 카운터에 기록 (자기 패턴 파악용)
- 휴식 세션은 통계에 별도 표시하지 않음 (집중 세션만 핵심 지표)

## 7. 통계

- **할일 단위**: 카드 우측에 누적 뽀모도로 횟수 표시 (예: 🍅 ×3)
- **일별 막대 그래프**: 최근 14일 또는 30일의 일별 완주 세션 수
- **요약 카드**: 오늘 / 이번 주 / 이번 달 완주 세션 수
- 그래프/통계 라이브러리는 가벼운 것으로 (구현 단계 결정)

## 8. 설정

- 집중 / 짧은 휴식 / 긴 휴식 시간 길이
- 긴 휴식 주기 (기본 4세션마다)
- 휴식 후 다음 집중 자동 시작 여부
- 알림 소리 on/off
- 데이터 내보내기/가져오기 (SQLite 파일 백업, 또는 JSON export — 미해결 의사결정 참조)

## 9. 비목표 (Out of Scope)

- 멀티 사용자, 계정/로그인
- 멀티 디바이스 실시간 동기화
- 모바일 네이티브 앱
- 협업/공유 기능
- 캘린더/외부 도구 연동

## 10. 성공 기준

- 할일을 추가하고, 클릭 한 번으로 25분 타이머가 시작된다
- 25분 동안 페이지를 새로고침해도 타이머가 정확히 복원된다
- 한 주가 지난 시점에 "이번 주 몇 뽀모도로 했는지", "어떤 할일에 가장 많이 썼는지" 한 화면에서 확인 가능하다
- 데이터는 SQLite 파일 하나에 모두 담겨, 그 파일만 백업하면 끝난다

## 11. 기술 결정사항

PRD 1차 범위에 적용할 기술 결정. 단순함을 최우선으로 선택했다.

- **백엔드**: Python 3.11+ / FastAPI / SQLAlchemy 2.x / SQLite (단일 파일). uvicorn으로 실행.
- **프론트엔드**: Vite + React 18 + TypeScript. 상태관리는 React 내장(useState/useReducer) + 필요 시 Zustand 정도까지만.
- **HTTP 클라이언트**: fetch 래퍼(직접 작성, 의존성 최소화).
- **배포/실행 방식**: 로컬 개발은 백엔드(uvicorn)와 프론트엔드(Vite dev) 두 프로세스. 배포는 Vite build 산출물(정적 파일)을 FastAPI가 함께 서빙하여 단일 프로세스로 실행.
- **데이터 백업 형식**: SQLite 파일 직접 복사(1차). JSON export는 비목표.
- **그래프 라이브러리**: Recharts (React 친화적, 가벼움).
- **태그 색상/UI**: 태그 이름 해시 기반 자동 색상 팔레트(고정 N색). 사용자 색 지정은 비목표.
- **단축키**: 미지원(1차 범위 외).
- **다크모드**: 미지원(1차 범위 외).
- **할일 정렬**: 우선순위 + 최근 활동(마지막 뽀모도로 시각) 기준 정렬만. 드래그 앤 드롭 수동 정렬은 비목표.
- **세션 메모**: 미지원(1차 범위 외).
- **테스트**: 백엔드는 pytest + httpx(TestClient). 프론트엔드는 Vitest + React Testing Library. E2E는 1차 범위 외.
- **린트/포맷**: 백엔드 ruff, 프론트엔드 ESLint + Prettier.
- **프로젝트 레이아웃**: 단일 저장소에 `backend/`(FastAPI)와 `frontend/`(Vite) 두 디렉터리 분리. SQLite 파일은 `backend/data/app.db`.
