# 할일 순서 변경 방식 (Evaluator 독립 선택)

선택: A

## 근거

PRD 3.1은 할일 관리를 추가/수정/삭제/완료 체크와 태그로 규정할 뿐
순서 변경을 언급하지 않으며, 확정 사항은 기본 정렬을 생성 순서
(`createdAt`)로 둔다는 점뿐입니다. PRD 6절이 v1 범위를 좁게 유지하는
기조를 감안하면, 순서 변경 기능은 핵심 가치(할일별 뽀모도로 집중 기록)에
직접 기여하지 않는 부가 기능입니다. 후보 B(위/아래 버튼)는 `order` 필드와
정렬 규칙 변경을, 후보 C(드래그 앤 드롭)는 추가 라이브러리 또는 HTML5 DnD
구현과 접근성 처리를 요구하여 비용 대비 우선순위가 낮습니다. 따라서 v1은
생성 순서 고정(A)이 적절하며, 순서 변경은 후속 버전 과제로 남깁니다.
