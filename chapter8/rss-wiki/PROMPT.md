# 당신의 임무

당신은 RSS Wiki 프로젝트를 구현하는 엔지니어입니다. 
PRD는 docs/PRD.md 에 있으며, 이 사양이 유일한 정답입니다. 
사양과 다르게 구현하거나 사양 자체를 슬쩍 바꾸지 마세요.

## 작업 절차

1. docs/TASKS.md 를 읽으세요. 없으면 PRD를 기반으로 새로 작성하세요.
   - 할일을 체크박스(- [ ])로 나열합니다.
   - 각 항목은 **한 세션(약 10~20분)** 안에 끝낼 수 있는 크기여야 합니다.
   - 의존성이 있는 항목은 위에서 아래로 순서대로.
2. TASKS.md 에서 완료되지 않은 첫 항목을 하나 고릅니다.
3. **그 항목 하나만** 구현합니다. 욕심내지 마세요.
4. 관련 테스트를 작성하고 `uv run pytest -x` 로 통과시킵니다.
5. 통과하면 해당 항목을 - [x] 로 바꿉니다.
6. docs/JOURNAL.md 맨 아래에 한 줄로 무엇을 했는지 기록합니다.
   형식: `- {ISO8601 타임스탬프} {한 문장 요약}`
7. 종료합니다.

## 제약

- 한 세션에서 TASKS.md 의 **한 항목만** 처리합니다.
- 코드는 src/rss_wiki/, 테스트는 tests/ 아래에 둡니다.
- 의존성 추가는 `uv add` 로만. pyproject.toml 직접 편집 금지.
- 막히면 해당 항목 아래에 `BLOCKED: {이유}` 라고 적고 종료하세요. 
  그 항목을 억지로 해결하려고 엉뚱한 파일을 뜯어고치지 마세요.

## 완료

PLAN.md 의 모든 항목이 - [x] 이면 프로젝트 완료입니다.
이 경우 아무 파일도 수정하지 말고 "완료" 한 줄만 남기고 바로 종료하세요.
